centos7服务器配置redis主从与哨兵

发表于 2020-05-11- 更新于 2020-05-12
-
online_course-linuxredis

  1. 构建centos7环境

    服务器构建centos7环境,没啥好写的,毕竟只需要在控制台操作一下就可以自动进行系统重装.

    这里要注意的是, 阿里云安装的centos7系统与正常你在自己虚拟机上安装的centos不同,在安装了图形化界面之后,使用init 5 和startX都是无法直接开启图形化界面的.

    这里需要进入阿里云控制台中,在远程连接选项中选择VNC连接, 这样才能进行图形化界面操作.

    值得一提的是,请务必记住你第一次进行VNC连接时生成的连接密码,它关系到你每次能否连接,如果你忘记了…..

    还是重装系统吧

  2. 配置python环境

    1. 这个的话,应该也没什么好说的吧,毕竟,centos本身自带了python环境(2.7)

    2. 如果有需要升级的话,可以参考下列操作

      wget  https://www.python.org/ftp/python/3.7.1/Python-3.7.1.tgz 
# 下载python 3.7.1 安装包
tar -zxvf Python-3.7.1.tgz  -C  /usr/local/
# 解压安装包
cd Python-3.7.1
# 切换路径至python文件夹
./configure --prefix=/usr/local/Python-3.7.1  --enable-shared CFLAGS=-fPIC
# 配置编译
make&&make install
# 编译源码,执行安装
python 3
# 查看是否安装成功

  3. 配置mysql数据库 + 主从复制

    这里可以参考我的博客: Centos7 环境下配置mysql主从服务

  4. 配置redis数据库 + 主从复制 + 哨兵集群

    这个问题我同样有另外一篇博客,就不在这里多说了: centos7服务器配置redis主从与哨兵集群

  5. 配置node环境

    wget https://nodejs.org/dist/v10.13.0/node-v10.13.0-linux-x64.tar.xz
# 下载nodejs安装包
tar xvf node-v10.13.0-linux-x64.tar.xz 
# 解压缩
mv node-v10.13.0-linux-x64 node-v10.13 
# 重命名
cd node-v10.13/bin
./node -v
# 进入目录查看其版本

    进入etc/profile 修改环境变量,使node命令能在任意目录运行:

    vim /etc/profile
//环境变量
NODE_HOME=/你的安装路径/node-v10.13
PATH=$NODE_HOME/bin:$PATH
export NODE_HOME
export PATH

    重启服务器: reboot
